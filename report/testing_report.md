# AI Resume Screening & Candidate Ranking System - Integration Testing Report

This document reports the integration testing workflow, execution results, test coverage, and edge-case handling for the AI Resume Screening & Candidate Ranking System.

---

## 1. Test Objective
The objective of integration testing is to verify that all modular components (Text Extraction, NLP Preprocessing, AI Similarity Calculation, and Candidate Ranking) interact correctly under the Streamlit execution thread. It validates that similarity scores correctly reflect candidate backgrounds and that invalid inputs do not cause runtime failures.

---

## 2. Test Environment
*   **Operating System:** Windows 10/11
*   **Python Version:** Python 3.10+
*   **Primary Packages tested:** PyPDF2 (v3.0.1), scikit-learn (v1.8.0), pandas (v3.0.3), streamlit (v1.57.0)

---

## 4. Findings & Summary
1.  **Ranking Validity:** The similarity scoring model behaves as expected. Job-specific profiles are matched with appropriate weighting. For instance, when looking for a Python Developer, the Python Dev resume scored ~31.3% while the Data Scientist scored only ~6.8%. The non-technical Sales profile scored a lower 2.8%, confirming the accuracy of TF-IDF vocabulary overlap checks.
2.  **Stability:** The UI remains interactive and fast. The file buffers from Streamlit's `file_uploader` are safely read into memory, cloned using `io.BytesIO`, and parsed without causing system locks.
3.  **UI Feedback:** Warnings are clearly rendered below the ranking dataframe, which alerts recruiters about empty or invalid candidate submissions without interrupting the general screening session.
